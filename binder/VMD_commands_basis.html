# VMD中的基本命令

作者: huanggang08

VMD Tcl/Tk中的基本命令

VMD中的基本操作，即能够熟练地使用Tcl语言是每一个学习计算机模拟的学生的基本功. 在介绍VMD中Tcl/Tk命令之前，我们要先了解Tcl的最基本的特征和其中的最基本的概念。

打开Tck控制台。打开VMD，在主窗口>Extensions>TkConsole,这就进入到了TkConsole 命令终端。所有的VMD里的操作，比如，改变原子颜色，改变背景，改变分子的角度，甚至制备样品等都可以做。

# 0. Tcl解释语言中的数据结构

# 0.1 Tcl的特征：

  1. 在Tcl中，所有的数据类型都被当作字符串(string)来操作.

# 0.2 一般语法：

command -flag -flag -flag <args> <args>

什么量可以做为参数值呢？是一些变量$variable, 它们可以是字符串，列表{}, 或者数组()。Tcl中的数组又称 hash-映射.

# 0.3 变量类性

# 0.3.0 字符串

我们来看，定义一个变量和打印一个变量的命令。

set a 1
puts $a

定义变量后，使用变量，前面要加$ 符号！

# 0.3.1 列表 (list)

定义一个列表：[2] [3]

set li {1 2 3 5}

或者

set li [list 1 2 3 5]

注意 [2]：从语法(形式)上看，与Python列表不同，Tcl列表中，元素之间无逗号。

[3] : Tcl/Tk控制台里，命令都是以绿色显示出来的。例如，下面例子中的set, list, puts等，都是命令，或者称为操作名

打印该列表：

puts $li

类似与Python里面的print()函数。所以，Tcl中的命令，就相当于函数，是对对象做一些操作。

再次注意，定义变量后，使用变量，前面要加$ 符号！ 其他命令(操作)有：

lindex: 提取元素 ; lsort： 对列表做排序操作; lset: 改变列表中某个元素的值.

puts [lindex $li 2]

puts [lsort -increasing $li]

lset li 2 0

最后一个命令的含义：将列表li的第三个元素设置为0. 注意：li 以字符串形式传入命令 lset，lset将该字符串理解为该字符串对应的列表。不是直接传入的列表li的值，所以，这里不加$符号。 下面的lappend 也是类似道理。

注意： 想打印出一个命令的结果，要用puts命令. 这和Anaconda里面Ipython交互环境，直接输入变量名就可以打印出其值不同。

还有，foreach循环：先来看一个一个地循环，

foreach x $li {puts $x}

foreach {x y} $li {puts “$x $y”}

我么当然也可以多个多个地循环。

“追加元素”操作：lappend

lappend li a

注意: 上例中，lappend的参数是字符串li.

lassign {1 2 3} a b c

将列表{1,2,3}中的 元素分别赋值给三个变量a,b,c.

lassign $li a b c d ##错的！##

# 0.3.2 Hash Maps/ Arrays/数组

Tcl中的数组，在其他语言里有别的名字. Awk中，它们被称为associative arrays, Perl 中被称为哈希映射(Hash maps), 在Python里，称为字典. [3]

注释[3] wiki.tcl-lang.org

为什么Hash映射如此重要呢？

原因：Tcl并不提供以数值索引的数组,Tcl数组的索引都是字符串！

Hash映射的特征：

Hash映射其实不应该称为是一种数据类型，而是一种与标量变量（scalar variables）相对的变量类型: array variables！ Hash映射是标量变量的集合.

“Only scalar variables have a value, each array element being a scalar variable.”

Tcl数组都是一维的。

不像列表，哈希映射不会保留它填充时的元素之间的顺序.

对Tcl数组操作有那些？

清除数组: 有多种方式, 如下三种：

array unset arr
unset arr
array set arr {}

# 1. 显示盒子

vmd版本： VMD1.9.4

# 1. 生成特定尺寸的模拟盒子

打开TCl/Tk (Tickle)控制台运行，输入命令

set cell [pbc set {15.64 15.64 31.28}]

set $variable

是定义一个变量，其名字为$variable.

这里的 cell是一个变量，它表示后面[]内命令的返回结果。你可以把它设置成其他名字，无所谓。例如，你可以这样写：

set x [pbc set {15.64 15.64 31.28}]

这个命令生成的模拟盒子和上面的是完全一样的，仅仅是名字不同而已。所以，set cell可以理解为生成一个名为cell的对象.

pbc 不是一个变量名, 而是命令. （绿色标识）

pbc set 表示要建立一个周期性盒子，{}内为三维矢量，作为pbc set的参数。 它表示盒子的大小。

这个命令，可以在打开文件后，在vmd终端运行；也可以在TCl/Tk控制台运行。执行了上面命令后，就有盒子显示出来了。 我们可以执行 打开和关闭pbc box 这样的简单操作。

pbc box -on

-on 参数是盒子显示出来. pbc box操作带上-off参数，就是消去盒子操作（让盒子不可见）。

pbc box -off

pbc box -on 和 pbc box -off其实可看做是同一个操作----"开关"操作。我们可以假设存在这样一个参数-swith, 那么， pbc box -switch on 等价与pbc box -on; pbc box -switch off 等价与pbc box -off.

# 另一种方法：

In vmd version 1.9, to show the periodic copies of a system:

  1. Open VMD main> Extensions> Tk Console
  2. In the VMD Tk Condole, type in
pbc set {a b c} -all
pbc box -on
  1. Graphics > Periodic Then select +X, -X, or any others.

注： 1. pbc set 作为一个整体，是一个设置周期盒子尺寸的命令; 2. pbc set 命令中的-all 参数表示要对所有时间点的构型都做同样的操作，即，定义一个周期性盒子. 如果不加-all,则该命令只在当前这一帧构型上操作！

# 2. 对盒子的状态进行操作

pbc box可以看成是作用在cell上的一个操作（函数），这个函数有几个参数，如颜色，中心，边线宽度等，加上这些参数后，pbc box可以更改盒子的颜色，更改盒子的中心位置，更改边线的宽度等。这些操作，使得cell这个对象，变到了一个新的状态.

设置盒子的中心位置：-center 是pbc box操作的一个参数，其参数值是com, 它操作的对象是当前的cell. 其他参数，如-color， -width等等也类似。

com: 指质心.

pbc box -center com

pbc box -center origin
pbc box -shiftcenterrel {1 0 0}

上面的{1 0 0}表示盒子移动的相对位置{a b c}的一个具体例子，表示一个矢量。设置盒子的边框的颜色：

pbc box -color black

pbc box -color red

改变边框的宽度：

pbc box -width 2.0

真正用的时候，是多个参数同时用：

pbc box -width 2.0 -center com -color black -width 2.0

# 3. 将原子都放到盒子中

我们可能会看到，模拟过程中，有些原子运动到盒子外面去了。坐标超出了盒子的范围。提出一个问题：有没有方法使原子的坐标限制在盒子里面呢？ 由于我们用的PBC, 所有，从右边出去的原子，其像会从盒子的左边再次进入。所以我们可以用原子的一个想到坐标代替原子的坐标。 我们来看看图形界面的操作，以此来显示原子的镜像。在主菜单>Graphics>Graphical Representations>Periodic 里面做选择。

pbc wrap就是实现的这个操作：使所有的原子的坐标都位于盒子之中。 wrap的本意是：围巾，包装材料 (n); 包裹，围起来(v)。

pbc wrap -center com

-center操作有其他参数，如bb, origin,...

将盒子的中心沿着x方向移动2个单位.

pbc wrap -shiftcenterrel {2,0,0}

命令中，{2,0,0}\{2,0,0\}{2,0,0} 表示三维空间的矢量。盒子的中心就沿着这个矢量移动一定的相对位移。

pbc wrap -all

注： 参数-all 的意思，与前面同理.

有时候，最小单位不能是原子，而必须是复合物。比如，分子，残基，分子链等.

pbc wrap -compound fragment

pbc wrap -compound residue

residue可简写为res, -compound参数其他值有chain,... (以残基为最小单位，还是以原子，还是以链为最小单位呢？ -compound就是在做这个选择。)

（跑一个分子动力学，演示wrap的效果。)

# 总结：

  1. 学一个命令/操作，首先要知道这个命令的总的效果，或者说它要实现的基本目标。
  2. 最好要理解这个命令的每一组成部分的本质。即要知道它的每一部分是什么。也就是说，要理解命令的结构/原理。就是说，基本概念要清楚，比如，命令和变量的区别，列表，Hash-Table.等.

例如

pbc box -center com

这个命令中，pbc box 是函数，其作用的对象(object)为cell, -center是pbc box这个操作(或者你可以称之为函数)的一个参数， com是参数-center中的具体值。注意有些参数不需要值，例如 -on, -off 等，就分别把pbc box 显示和隐藏。可以认为他们分别是某个隐藏参数switch的值.
