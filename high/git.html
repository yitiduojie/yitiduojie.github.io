主要内容：

# 第1章讲的是命令行简介，如Shell等.

准确地查找出需要的文件对于我提高工作效率往往非常重要．在Shell中设置环境变量可以在这点上帮我一把．我学会了将常用的目录设为环境变量，保存在.bashrc里面．例如，添加如下一行命令

\texttt{export SFG=/home/huang/P1--SFG_using_Propagator}

到.bashrc文件中, 下次到\texttt{/home/huang/P1--SFG_using_Propagator}目录就只需要输入

\texttt{cd \$SFG}

查找文件变得更加简单.

# 第15章讲的是版本控制工具Git.

我学会了下面几个命令.

Git基本：

gitinit\text{git init}gitinit

\texttt{git add } : 将文件<filename>添加到代码库中

\texttt{git status} : 检查本地副本的状态.　已提交，还是未提交

\texttt{git commit -m "some statements"}: 　提交版本并写入作者想要的说明

\texttt{git log}: 查看历史．尤其可以查看到每次提交的标识号

\texttt{git diff}: 查看版本之间的差异

关于分支管理：

\texttt{git branch}: 查看所有分支及当前所在分支．

\texttt{git branch simple}: 创建一个名为simple的分支

\texttt{git branch -d simple}: 如发现该分支不再需要时，可以用\texttt{-d} 选项删除该分支.

\texttt{git checkout simple}: 进入\texttt{simple}分支

关于合并分支：

\texttt{git checkout master};

\texttt{git merge simple}: 这两个命令实现的功能是将simple分支与主干合并.

关于回滚的操作：

\texttt{git reset 37de7de1} : 退回到某个文件版本．文件以标识号（散列值）标识，或以标识号的前８位标识

\texttt{git reset [] []}: 　mode 可选项有三种可能的值：　\texttt{--soft, --mixed, --hard}.

\texttt{git revert }: 永久性地删除某个版本，不留记录痕迹．用于删除不小心提交私人信息.
